# 太陽光発電・日射量の用語と計算

このプロジェクトを触るうえで知っておくと迷わない用語と式をまとめる。

## 1. 日射量の3成分

| 略 | 正式名 | 何の値か | 単位 |
|---|---|---|---|
| **GHI** | Global Horizontal Irradiance | 水平面が受ける全天日射量 | W/m² |
| **DNI** | Direct Normal Irradiance | 太陽に正対した面の直達日射量 | W/m² |
| **DHI** | Diffuse Horizontal Irradiance | 水平面が受ける散乱日射量 | W/m² |
| **GTI** | Global Tilted Irradiance | 傾いたパネル面が受ける全天日射量 | W/m² |
| (POA) | Plane of Array | GTI と同義（PV業界の言い回し） | W/m² |

### 関係式

```
GHI = DNI × cos(θz) + DHI       (θz は太陽天頂角)
GTI = (DNI/DHI/反射成分をパネル面に再投影して合成)
```

### 単位の使い分け

- **瞬時値**: W/m²（"いまこの瞬間"の強さ）
- **積算値**: kWh/m² = 1時間あたり W/m² の積分。"今日の合計"
- 例: 12時に 800 W/m²、1日合計で 5 kWh/m²、年間で 1300 kWh/m² (本州内陸の典型)

---

## 2. それぞれをどこで使うか

| 値 | 主な用途 | 代表ソース |
|---|---|---|
| **GHI** | その地点の日射ポテンシャル | NEDO 平年値、NASA POWER、JMA 全天日射量、CAMS |
| **GTI** | 実際のパネル発電量に直結 | Open-Meteo（tilt/azimuth 指定で内部計算）、Solcast |
| **DNI** | 集光型太陽熱発電 (CSP)、追尾型 | Solcast、Solargis |

---

## 3. パネル幾何の決め打ち（このプロジェクト）

[app.js](../dashboard/app.js) の定数:

```js
const PERFORMANCE_RATIO = 0.80;   // 損失込みの実効係数
const TILT_DEG = 30.0;            // パネル傾斜角。0=水平
const AZIMUTH_DEG = 0.0;          // 方位角。0=真南、-90=東、90=西
```

- **TILT = 30°、AZIMUTH = 0°（南向き）** は日本の住宅・産業用PVで標準的な仮定
- 施設ごとの実際のパネル幾何は FIT 公表データには含まれないため、全国一律で決め打つのが現実解
- 厳密な傾斜最適値は緯度に応じ 25〜35° 程度（高緯度ほど立てる）

## 4. 発電量計算（このプロジェクトの式）

### 積算（当日/翌日モード）

```
推定発電量 [kWh]
  = 設備容量 [kW]
  × Performance Ratio (= 0.80)
  × その日の GTI 積算 [kWh/m²]
```

### 瞬時（現在モード）

```
推定出力 [kW]
  = 設備容量 [kW]
  × Performance Ratio
  × (現在の GTI [W/m²] / 1000)
```

`/1000` は「STC (Standard Test Conditions) = 1000 W/m² 時にカタログ定格出力が出る」という定義から。

---

## 5. Performance Ratio (PR)

- 「パネルが受けた日射エネルギーのうち、電力として取り出せた割合」
- 損失要因: パワコン変換損、配線損、温度損、汚れ、影、ミスマッチ
- **国内の実機 PR は 0.75〜0.85 が典型**。0.80 は穏当な平均値
- 夏は温度損で下がり、冬は上がる傾向（**温度補正していない簡易モデル**）

---

## 6. Capacity Factor (CF, 設備利用率)

```
CF = 実発電量 [kWh] / (設備容量 [kW] × 24 h)
```

- 「もしフル稼働し続けたら出せたエネルギーに対する実績の比」
- 国内 PV の年間 CF はおおよそ **13〜17%**
- 太陽光は夜があるので CF が低くなるのが宿命（風力 25〜35%、原子力 70〜90% と比較）
- **本プロジェクトでは「現在モード」のとき "出力率 = 瞬時出力 / 容量" を CF と同じ枠で表示している** — 厳密には別物だがUI簡略化のため

## 7. GHI → GTI の変換（参考）

雲量ベースの代替案で必要になる変換。簡易・厳密の段階:

### 超簡易（緯度近似）

```
GTI ≈ GHI × cos(latitude − tilt)
```

精度は粗いが「概算」用途には十分。

### 中精度（Hay-Davies モデル）

```
GTI = DNI × cos(入射角)
      + DHI × (異方性散乱項)
      + GHI × アルベド × (1 − cos(tilt)) / 2
```

太陽位置（高度・方位角）が必要 → **SPA** または **suncalc** で計算。

### 高精度（Perez モデル）

雲の影響まで考慮、商用ソフトの標準。実装は重い。

---

## 8. クリアスカイモデル

「雲が無いときの理論 GHI」を求める式。雲量補正と組み合わせて使う。

| モデル | 特徴 |
|---|---|
| **Ineichen-Perez** | 大気透過率 (Linke turbidity) を入力。よく使われる |
| **Bird** | 大気成分（オゾン・水蒸気・エアロゾル）を入力。透明 |
| **ESRA** | 欧州標準、CAMS の内部実装 |
| **Haurwitz** | 太陽高度だけで近似。**1行で書ける簡易版** |

最小実装:

```
GHI_clear ≈ 1098 × cos(θz) × exp(-0.057 / cos(θz))     (Haurwitz)
```

## 9. 日射量を間接推定する手法（直接得られないとき）

日射計データや GTI 直配信 API（Open-Meteo、Solcast 等）が使えない場合に、別の観測値・派生プロダクト・実績値から GHI/GTI を推定する手段の整理。**API 障害時のフォールバック設計** および **実測校正（PR テーブル）の根拠** として使う。

### 9.1 雲量ベース（経験式）

最も使われる近似。クリアスカイ GHI（[§ 8](#8-クリアスカイモデル)）に雲量補正をかける。

```
GHI ≈ GHI_clear × (1 − k × CloudCover / 100)        (線形, k=0.6〜0.75)
GHI  = GHI_clear × (1 − 0.75 × (CC/100)^3.4)        (Kasten-Czeplak, 非線形)
```

- 本プロジェクトでは **Open-Meteo の `cloud_cover` を入力**にこれを適用するのが第一候補
- 雲量だけだと光学厚さや雲種が反映されない → 精度上限あり（R² ≈ 0.7–0.85）

### 9.2 日照時間ベース（アンクストローム=プレスコット式）

「日照時間 / 可照時間」から GHI を推定する古典式。**アメダスや気象庁観測点では日照時間がリアルタイム取得可能**なので、雲量取得が途絶しても代替できる。

```
GHI = GHI_clear × (a + b × n / N)        (n=実日照時間, N=可照時間)
```

- `a ≈ 0.25`, `b ≈ 0.5` が地域平均。日本向けに地域校正可
- 精度: R² ≈ 0.7–0.9（雲量式より頑健なことが多い）
- アメダスは CORS 不可 → サーバー前処理 or 静的 JSON 化が必要

### 9.3 気温日較差ベース（Bristow-Campbell / Hargreaves-Samani）

**気温の最高/最低（ΔT = Tmax − Tmin）から GHI を推定**する。日射計も日照時間計もない地点でも、気温データだけで動くのが強み。

```
GHI = GHI_clear × (1 − exp(−B × ΔT^C))          (Bristow-Campbell)
GHI = K_rs × GHI_clear × sqrt(ΔT)               (Hargreaves-Samani, K_rs ≈ 0.16–0.19)
```

- 精度: R² ≈ 0.5–0.8。雲量式に劣るが、入力欠損に強い
- FAO の灌漑分野で標準化されており、地域校正係数の文献値が多い

### 9.4 衛星画像派生（実況〜超短期）

| ソース | 単位 | 取得 | 即時性 |
|---|---|---|---|
| ひまわり8/9 標準データ (JAXA P-Tree) | 雲量・派生 GHI | 要登録・無料 | 10分間隔 |
| NICT サイエンスクラウド ひまわりWeb | 可視/赤外画像 | 無料 | 10分間隔 |
| CAMS Radiation Service (SoDa) | GHI/DNI/DHI 直接 | 要登録・無料 | 1日遅延 |
| NASA POWER | GHI 日次/時間 | 無料・無キー・**CORS可** | **履歴のみ・予報なし** |

CAMS と NASA POWER は **校正の真値** として使うのが定石。リアルタイム性は弱い。

### 9.5 数値予報モデルの別窓口（Open-Meteo 以外）

| ソース | 短波放射 | 解像度 | 備考 |
|---|---|---|---|
| ECMWF Open Data | SSRD あり | 0.25° | GRIB2、CORS △ |
| NOAA GFS (NOMADS) | DSWRF あり | 0.25° | GRIB2 |
| DWD ICON Open Data | 短波あり | 0.125° | GRIB2 |
| **Open-Meteo の `models=` 切替** | GTI 直接 | — | **同 API で ECMWF/GFS 取得可。429 時の自然なフォールバック先** |

→ Open-Meteo の `models=ecmwf_ifs025,gfs_seamless` を `jma_seamless` の代替として使うのが、最も低コストな多重化。

### 9.6 実発電量からの逆算（真値に最も近い）

`実発電量 ÷ 容量 ÷ PR ≒ 実 GHI(GTI)` という関係を使う。**過去データの後付け校正**に強い。

| ソース | 即時性 | CORS |
|---|---|---|
| OCCTO 需給実績 | 5–30分遅延 | × |
| 電力10社 太陽光発電実績 | 5–30分遅延 | × |
| PVOutput.org（コミュニティ実測） | 即時 | △ |

**本プロジェクトでの最有力用途**: [§ 5](#5-performance-ratio-pr) の「PR=0.80 一律」を地域別・季節別に校正するためのバッチ前処理データ源。

### 9.7 補助データを使う簡易フォールバック

| 手段 | 説明 | 用途 |
|---|---|---|
| 降水ナウキャスト逆利用 | 「雨レーダー反応なし」≒「雲少なめ」のラフな代理 | 雲量取得が完全に途絶した非常時 |
| 天気予報テキスト → 雲量変換 | 「晴」=10%、「曇」=80% など対応表 | bosai JSON から雲量を間接抽出 |
| NEDO METPV-20 月別平年値 | 県別 12 ヶ月分のクライアントサイド同梱 | **全 API 死亡時の最終フォールバック**。数 KB |
| 気象庁 気候平年値（日照時間） | アメダス約1300点 | アンクストローム式（[§ 9.2](#92-日照時間ベースアンクストロームプレスコット式)）の入力源 |

### 9.8 ハイブリッド・優先順位（本プロジェクトでの推奨）

1. **多重化**: Open-Meteo `models=` で MSM/GSM + ECMWF + GFS を併用 → 429 時の自動フォールバック
2. **最終防衛線**: NEDO METPV-20 県別×月別 GHI 平年値を JSON 同梱
3. **PR 校正**: OCCTO の太陽光実績をバッチで取得・静的 JSON 化 → 地域別 PR テーブル生成
4. **検証用**: CAMS Radiation Service を Python バッチで毎日取得し過去 GHI 蓄積
5. **やらない**: 自前のひまわり画像解析・天気テキストパーサ（実装コスト > 効果）

平年値（NEDO）+ 偏差（雲量・実績）の 2 項分解で「平均は外さない・異常時のみ予報モデルに頼る」構成が、最小実装で最大の頑健性を出せる。

## 10. 太陽位置（座標系）

- **太陽高度 (Altitude / Elevation)**: 地平線から太陽までの角度。90° = 真上
- **太陽天頂角 (Zenith)**: 90° − 太陽高度
- **太陽方位角 (Azimuth)**: 真北を0、東90、南180、西270 とする流派と、真南を0とする流派がある（**Open-Meteo は真南=0**、`suncalc` は真南=0）

## 11. 関連単位の換算

| 換算 | 値 |
|---|---|
| 1 kW = 1000 W | — |
| 1 MW = 1000 kW | — |
| 1 kWh = 1 kW × 1 hour | エネルギー |
| 1 MWh = 1000 kWh | — |
| 一般家庭の年間電力消費 | ≈ 4,500 kWh |
| 住宅 PV 4 kW システムの年間発電量 | ≈ 4,500 kWh |

---

## 12. ありがちな取り違え

- **kW (出力 = パワー) と kWh (発電量 = エネルギー)** は別物。混同しがち
- **設備容量 (kW)** はパネル定格の合計。実発電量は気象と PR に依存
- **GHI と GTI** はパネルが水平か傾いているかで使い分け
- **方位角の流派**: API ごとに 0=北 / 0=南 が違うので必ず仕様確認
- **MWh と MW**: 「○○MWの太陽光発電所」は容量、「○○MWh発電」はエネルギー
